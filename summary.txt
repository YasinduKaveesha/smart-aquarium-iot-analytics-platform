SECTION 1 — WHAT THIS SYSTEM DOES (NON-TECHNICAL)
────────────────────────────────────────────────────────────────────────────────

Imagine a smart health monitor for your fish tank — like a smartwatch for your
aquarium. Most basic systems only tell you "the water is bad RIGHT NOW." By the
time you get that alert, your fish are already stressed.

This system is different. It does three things a basic system cannot:

  1. PREDICTS THE FUTURE
     Using a forecasting model (SARIMA), the system looks 60 minutes ahead and
     asks: "If the water keeps trending this way, will it become dangerous?"
     It uses the WORST-CASE edge of its prediction — the most pessimistic
     scenario — to calculate the Water Quality Index (WQI).

  2. UNDERSTANDS GRADUAL DANGER (FUZZY LOGIC)
     A basic system treats pH 6.50 as "fine" and pH 6.49 as "critical."
     This system has no cliff-edge. As values drift toward danger zones,
     the health score smoothly decreases — just like real biological stress
     builds gradually in fish.

  3. KNOWS WHEN SOMETHING IS WRONG WITH THE SENSORS
     Using an Isolation Forest AI model, the system detects if a sensor
     reading is an unusual spike or failure. When an anomaly is detected,
     it reduces its confidence in the WQI by 20% automatically.

The result: you get a warning BEFORE the water becomes dangerous, not after.
Measured lead time advantage: 5 minutes early warning for step-change anomalies,
with a 60-minute theoretical forecast horizon for gradual drifts.

└─────────────────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  LAYER C — Forecasting Engine (Future Intelligence)                         │
│  Model: SARIMA (Seasonal ARIMA) — two models, one per parameter             │
│                                                                             │
│  pH   Model: SARIMAX(1,1,0)(0,0,1,24)                                      │
│    d=1 handles pH non-stationarity; MA(1) dropped (was insignificant p=0.54)│
│    S=24 captures the 24-hour biological rhythm of the tank                  │
│    MAE (in-sample): ~0.08 pH units                                         │
│                                                                             │
│  Temp Model: SARIMAX(1,0,1)(0,1,0,24)                                      │
│    D=1 handles seasonal drift; seasonal MA dropped (was degenerate at -1.0) │
│    S=24 captures daily heater/lighting cycles                               │
│    MAE (in-sample): ~0.39 °C                                               │
│                                                                             │
│  KEY INNOVATION — Asymmetric Risk Bounds (95% Confidence Interval):        │
│    pH   → LOWER 95% CI bound  (worst-case: pH could drop this low)         │
│    Temp → UPPER 95% CI bound  (worst-case: temp could rise this high)      │
│    Rationale: different biological risks require different pessimistic edges│
│    pH lower CI: ±0.0505 below mean  |  Temp upper CI: ±0.4895 above mean  │
└─────────────────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  LAYER D — Adaptive WQI Fusion (Interpretation)                             │
│  Species Profile: Neon Tetra (Paracheirodon innesi)                        │
│                                                                             │
│  Fuzzy Membership Functions (trapezoidal, numpy-based):                    │
│    pH   : ideal 6.0–7.0, warning 5.5–6.0 / 7.0–7.5, critical <5.5 / >7.5 │
│    Temp : ideal 22–26°C, warning 20–22 / 26–28°C, critical <20 / >28°C   │
│    TDS  : ideal 50–150 ppm, warning 150–200 ppm, critical >200 ppm        │
│    Turb : ideal 0–5 NTU, warning 5–10 NTU, critical >10 NTU               │
│                                                                             │
│  Compound Stress Penalty:                                                   │
│    When pH is low AND temperature is high simultaneously, biological        │
│    stress is amplified (osmoregulatory + metabolic synergy).               │
│    Formula: penalty = 1 - (0.30 × pH_deviation × Temp_deviation)          │
│    Range: [0.70, 1.0] — maximum 30% additional penalty                    │
│                                                                             │
│  Final Formula:                                                             │
│    WQI_Adaptive = FuzzyScore(SARIMA_PessimisticCI)                         │
│                 × CompoundStressPenalty                                     │
│                 × AnomalyConfidence (0.80 if anomaly, 1.0 if normal)       │
│                                                                             │
│  Weights: pH=35%, Temp=25%, TDS=20%, Turbidity=20%                        │
└─────────────────────────────────────────────────────────────────────────────┘

  notebooks/03_forecasting_model.ipynb
  ─────────────────────────────────────
  Purpose : Train SARIMA models for pH and temperature forecasting
  Training: Hourly resampled data (720 hourly points from 5-min data)
  Train/test split: 80% train (576h), 20% test (144h)

  pH Model — SARIMAX(1,1,0)(0,0,1,24):
    Design choice: d=1 only; MA(1) dropped (insignificant p=0.54); seasonal AR dropped
    Seasonal MA(24) retained — captures 24h CO2/photosynthesis cycle
    MAE on test set: ~0.08 pH units

  Temp Model — SARIMAX(1,0,1)(0,1,0,24):
    Design choice: D=1 for seasonal drift; seasonal MA dropped (was degenerate ma.S.L24=-1.0, p=0.995)
    D=1 alone sufficient to capture seasonal pattern cleanly
    MAE on test set: ~0.39 °C

  Early Warning Demo: 24-hour ahead health check
    All hours predicted stable → "Status: All parameters predicted stable"
    (Tank conditions stable at end of training window)

  Saved artifacts:
    models/sarima_ph_v1.pkl    (statsmodels native .save() format)
    models/sarima_temp_v1.pkl  (statsmodels native .save() format)
    NOTE: .pkl not .joblib — statsmodels objects must use their own serialisation

  notebooks/04_adaptive_wqi.ipynb
  ────────────────────────────────
  Purpose : Implement and evaluate the full Adaptive WQI pipeline
  Depends : sarima_ph_v1.pkl, sarima_temp_v1.pkl, isolation_forest_v1.joblib,
            scaler_v1.joblib

  Key cells:
    imports_cell       : Load all artifacts, verify feature_cols = 4 items
    fuzzy_cell         : Define + visualise trapezoidal membership functions
    compound_cell      : Define + visualise compound stress surface (pH × Temp)
    sarima_forecast_cell: 12-hour ahead forecast with 95% CI table
    ci_viz_cell        : 2-panel plot of in-sample CI bands (first 7 days)
    compute_wqi_cell   : Pre-compute CI pessimistic bounds → run 8,628-row loop
    viz_main_cell      : 3-panel comparison plot
    leadtime_cell      : Lead time quantification
    save_cell          : Save wqi_df to CSV

  Results:
    Rule-Based WQI mean : 47.2  |  Min: 2.0   |  Std: 22.05
    Adaptive WQI mean   : 46.2  |  Min: 5.6   |  Std: 22.03
    Lead Time Advantage : 5 minutes (first adaptive < 70 at 03:30,
                          first rule-based < 70 at 03:35)
    Anomaly rate        : 6.0% (519/8,628)
    Fuzzy floor effect  : Adaptive min=5.6 vs Rule-Based min=2.0
                          (fuzzy scoring prevents catastrophic cliff-edge drops)

  Why mean gap is ~1 point (honest analysis):
    Actual pH mean = 7.0045 (just above 7.0, the Neon Tetra ideal zone upper limit)
    pH lower CI = 6.9720 — this is INSIDE the ideal zone → gets membership = 1.0
    Actual pH 7.0045 is slightly above ideal → membership = 0.991
    So CI-based "pessimistic" pH is actually slightly more ideal than actual pH
    The gap appears during anomaly events (hence 5-min lead time exists)
    This is a dataset characteristic, not a code flaw.

  Saved output:
    models/adaptive_wqi_results.csv

SECTION 4 — PRODUCTION PIPELINE (src/ files)
────────────────────────────────────────────────────────────────────────────────

The src/ directory contains production-ready, importable Python modules.
These are what a backend API endpoint would import and call.

  src/anomaly_detection_pipeline.py     [Pipeline 02]
  ─────────────────────────────────────────────────────
  Class  : AnomalyDetector
  Method : predict(temperature, ph, tds, turbidity) → (anomaly_flag, anomaly_score)
  Loads  : isolation_forest_v1.joblib, scaler_v1.joblib
  Note   : Uses DataFrame with named columns to match scaler's fitted feature names

  src/forecasting_pipeline.py           [Pipeline 03]
  ─────────────────────────────────────────────────────
  Class  : SARIMAForecaster
  Methods:
    get_forecast_ci(steps=12)
      → (ph_ci_df, temp_ci_df)  — full CI tables for reporting
    get_pessimistic_forecast(steps=1)
      → (ph_lower_series, temp_upper_series)  — for live WQI input
    build_insample_pessimistic_5min(df_index, forecast_steps=25)
      → (ph_lower_5min, temp_upper_5min)  — for batch historical analysis
      Note: dynamic indexing — works for any dataset length
  Loads  : sarima_ph_v1.pkl, sarima_temp_v1.pkl (SARIMAXResults.load())
  Handles: 26-hour SARIMA warmup artefact via bfill()

  src/adaptive_wqi_pipeline.py          [Pipeline 04]
  ─────────────────────────────────────────────────────
  Functions (standalone, stateless):
    trapezoidal_membership(x, a, b, c, d)
    ph_membership(x), temp_membership(x), tds_membership(x), turbidity_membership(x)
    compound_stress_penalty(ph, temp) → float in [0.70, 1.0]
    rule_based_wqi(ph, temp, tds, turbidity) → float in [0, 100]
    adaptive_wqi(current_vals, forecasted_vals, anomaly_flag)
      → (score: float, breakdown: dict)
  Class  : AdaptiveWQIPipeline (thin wrapper with .rule_based() and .compute())
  Note   : Fully stateless — no model loading — runs in microseconds

  src/sensor_validator.py               [Upgrade 4 — Input Validation]
  ─────────────────────────────────────────────────────────────────────
  Class  : SensorValidator
  Method : validate(ph, temperature, tds, turbidity) → ValidationResult
  Checks : None/NaN, below physical minimum, above physical maximum
  Bounds :
    pH          : 0.0 – 14.0  (full acid-base scale)
    temperature : 0.0 – 50.0°C
    TDS         : 0.0 – 2000 ppm
    turbidity   : 0.0 – 500 NTU
  Returns: ValidationResult(is_valid: bool, errors: list[str])
  Purpose: Catches disconnected or malfunctioning sensors before they reach
           the AI model and produce silent wrong predictions

  src/pipeline_router.py                [Upgrade 3 — Cold-Start Guard]
  ─────────────────────────────────────────────────────────────────────
  Class  : AquariumPipelineRouter(install_date: date)
  Method : route(ph, temperature, tds, turbidity) → RouterResult

  Routing Logic (in priority order):
    1. If sensor validation fails   → mode='SENSOR_ERROR'
       Returns rule-based fallback WQI + list of hardware faults
    2. If tank_age < 14 days        → mode='COLD_START'
       Returns rule-based WQI + "X days until Adaptive WQI unlocks"
    3. If tank_age >= 14 days       → mode='ADAPTIVE'
       Returns adaptive WQI + full breakdown + anomaly flag

  RouterResult fields:
    mode                : 'COLD_START' | 'ADAPTIVE' | 'SENSOR_ERROR'
    wqi_score           : float (0–100)
    anomaly_flag        : int (0=normal, 1=anomaly, -1=sensor error)
    days_until_adaptive : int (0 when adaptive is active)
    breakdown           : dict (component scores for dashboard display)
    sensor_errors       : list[str] (empty unless SENSOR_ERROR)

  src/retrain_sarima.py                 [Upgrade 1 — Rolling Retraining]
  ─────────────────────────────────────────────────────────────────────
  Function: retrain_sarima(data_csv, model_dir, lookback_days=30)
  Purpose : Re-fit SARIMA models on the most recent N days of live sensor data
  Steps   :
    1. Load CSV → resample to 1H mean
    2. Slice last lookback_days of data (default 30 days)
    3. Fit SARIMAX(1,1,0)(0,0,1,24) on pH
    4. Fit SARIMAX(1,0,1)(0,1,0,24) on Temp
    5. Save with .save() → overwrites sarima_ph_v1.pkl, sarima_temp_v1.pkl
    6. Reports in-sample MAE for both models
  CLI     : python -m src.retrain_sarima --data <csv> --models <dir> --days 30
  Schedule: Run weekly via cron job to prevent model staleness
  Verified: pH MAE~0.08, Temp MAE~0.39 on simulation dataset

SECTION 5 — DEPLOYMENT LIFECYCLE (The 14-Day Roadmap)
────────────────────────────────────────────────────────────────────────────────

  DAY 0 — Setup
  ─────────────
  User action  : Clean tank, perform water change, install sensors
  System state : All pipelines loaded, router initialised with install_date=today
  Why important: Establishes the "clean baseline" for the AI to learn.
                 The Nitrogen Cycle has not yet started.

  DAYS 1–14 — Cold Start / Calibration (mode='COLD_START')
  ─────────────────────────────────────────────────────────
  User experience : WQI shows values, but no predictive alerts
  System state    : Rule-Based WQI active for immediate safety monitoring
                    SARIMA models observing the tank's diurnal rhythm
                    Isolation Forest running in the background
  Dashboard shows : "Calibrating... X days until Adaptive WQI unlocks"
  Why this exists : A freshly set-up tank goes through the Nitrogen Cycle.
                    Ammonia and nitrite spike before beneficial bacteria
                    establish. The SARIMA model must first learn the tank's
                    unique 24-hour biological and lighting rhythm before
                    its predictions are trustworthy.

  DAY 15+ — Adaptive Intelligence (mode='ADAPTIVE')
  ───────────────────────────────────────────────────
  User experience : Full predictive alerts active
  System state    : All 4 layers active
                    SARIMA generates 95% CI pessimistic bounds
                    WQI uses forecasted worst-case pH and temperature
                    Anomaly confidence weight applied automatically
  Lead time       : 5 minutes verified | up to 60 minutes theoretical horizon
  Dashboard shows : Adaptive WQI score, anomaly status, full breakdown

  WEEKLY — Rolling Retraining
  ────────────────────────────
  Trigger : Cron job or manual run of src/retrain_sarima.py
  Action  : Re-fits SARIMA on last 30 days of live data
  Effect  : CI bands stay tight and accurate as tank conditions evolve
  Command : python -m src.retrain_sarima --days 30

SECTION 10 — KNOWN LIMITATIONS & HONEST ASSESSMENT
────────────────────────────────────────────────────────────────────────────────

  1. LEAD TIME IS 5 MINUTES, NOT 15–30
     The 5-minute lead time is real and verified. Claims of 15–30 minutes
     would require gradual-drift anomalies in the dataset. The simulation
     dataset uses step-change anomalies (sudden pH drops), which limits
     the observable lead time. The architecture supports longer lead times
     for gradual drift — the dataset does not demonstrate it.

  2. MEAN WQI GAP IS ~1 POINT
     Adaptive mean=46.2 vs Rule-Based mean=47.2 — 1-point difference.
     Root cause: Dataset pH (mean 7.0045) hovers just above the Neon Tetra
     ideal zone upper limit (7.0). The lower CI (6.97) falls inside the
     ideal zone and scores better than actual pH. This cancels the Temp
     upper CI penalty for most readings. The gap appears during anomaly events.

  3. SARIMA MODELS TRAINED ON SIMULATION DATA
     The current models are fitted on synthetic/simulated sensor data.
     In production, models should be retrained on real tank data after
     the 14-day calibration period using src/retrain_sarima.py.

  4. LDR (LIGHT SENSOR) NOT IMPLEMENTED
     The architecture concept mentions LDR for circadian-aware thresholds.
     This is a planned future enhancement — not in the current feature set.
     LDR would allow the system to distinguish natural nocturnal pH drops
     from actual acid crashes.
